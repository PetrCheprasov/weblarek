# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Vite

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

#### IProduct (Товар)
Данный интерфейс задает структуру объекта, описывающего товар в системе. Он служит для стандартизации данных и их удобного использования в различных частях приложения.

Элементы интерфейса:

`id: string` — Уникальный строковый идентификатор, присваиваемый товару на стороне сервера.

`title: string` — Наименование товара, отображаемое в его карточке.

`image: string` — URL-адрес изображения для карточки товара.

`category: enum string` — Принадлежность товара к одной из предопределенных категорий: хард-скил, софт-скил, кнопка, дополнительное, другое.

`price: number | null` — Стоимость товара. Возможность иметь значение null реализована для управления отображением кнопки покупки (например, скрывать её, если цена не указана).

`description: string` — Текстовое описание товара, которое пользователь видит при открытии подробной карточки.

#### Интерфейс IBuyer (Покупатель)
Этот интерфейс определяет формат хранения информации о клиенте, оформившем заказ.

Элементы интерфейса:

`payment: 'cash' | 'card' | '' `— Выбранный способ расчета: наличными при получении, банковской картой или неопределенное состояние.

`address: string`— Адрес доставки заказа.

`email: string` — Адрес электронной почты для связи.

`phone: string` — Номер телефона покупателя.

#### Модели данных (Data Models)

#### Класс Catalog (Каталог)
Данный класс представляет собой центральную модель для работы с каталогом товаров. Он отвечает за хранение массива товаров и управление выбранным в данный момент товаром, используя интерфейс IProduct.

Внутреннее состояние (Поля):

`protected products: IProduct[]` — Массив, содержащий загруженные с сервера данные о всех товарах.

`protected pickedProduct: null | IProduct` — Ссылка на товар, который в настоящий момент выбран пользователем для просмотра деталей или добавления в корзину.

Функциональность (Методы):

`public getProducts(): IProduct[]` — Возвращает полный массив всех товаров, хранящихся в модели.

`public setProducts(products: IProduct[]): void` — Загружает переданный массив товаров во внутреннее хранилище.

`public getProductByID(id: string): IProduct | null` — Выполняет поиск товара по его уникальному идентификатору. Возвращает найденный товар или null.

`public getPickedProduct(): IProduct | null` — Предоставляет данные о текущем выбранном товаре.

`public setPickedProduct(product: IProduct): void` — Устанавливает переданный товар в качестве выбранного.

#### Класс Cart (Корзина товаров)
Класс Cart инкапсулирует логику работы с корзиной покупок. Для хранения товаров используется коллекция Set, что гарантирует уникальность каждого товара в списке.

Внутреннее состояние (Поля):

`protected cartItems: Set<IProduct>` — Коллекция, хранящая уникальные товары, добавленные в корзину.

Функциональность (Методы):

`public addProduct(product: IProduct): void` — Добавляет новый товар в коллекцию корзины.

`public deleteProduct(product: IProduct): void` — Удаляет указанный товар из корзины.

`public clearCart(): void` — Полностью очищает корзину.

`public getQuantityCartItems(): number` — Подсчитывает и возвращает общее количество позиций в корзине.

`public getCartItems(): IProduct[]` — Возвращает массив всех товаров, находящихся в корзине.

`public getTotalPrice(): number` — Вычисляет итоговую стоимость всех товаров в корзине.

`public cartHasProduct(product: IProduct): boolean` — Проверяет, присутствует ли конкретный товар в корзине, возвращая true или false.

#### Класс Buyer (Покупатель)
Эта модель предназначена для работы с данными о покупателе. Она включает механизм валидации введенной информации перед её использованием.

Конструктор:

`constructor(payment: '' | "cash" | "card", address: string, email: string, phone: string)` — Создает новый экземпляр покупателя, инициализируя его поля переданными значениями.

Внутреннее состояние (Поля):

`protected buyer: IBuyer` — Объект, содержащий всю информацию о покупателе, соответствующую интерфейсу IBuyer.

Функциональность (Методы):

`protected checkData(): boolean` — Внутренний метод для проверки корректности и полноты данных. Возвращает true, если данные валидны.

`public getData(): IBuyer` — Возвращает объект с данными о покупателе.

`public setData(newBuyer: IBuyer): void` — Заменяет текущие данные покупателя на новые, переданные в метод.

#### Слой взаимодействия с сервером (Communication Layer)

#### Класс ComApi
Данный класс выступает в роли адаптера или фасада для низкоуровневого класса Api. Он не наследует, а использует его функциональность через композицию (внедрение зависимости), предоставляя более удобные и безопасные методы для обмена данными с сервером, включая проверку их структуры.

Конструктор:

`constructor(api: IApi)` — Принимает экземпляр класса, реализующего интерфейс IApi, и сохраняет его для дальнейшего использования.

Внутреннее состояние (Поля):

`protected api: IApi` — Ссылка на объект, который предоставляет базовые методы для выполнения HTTP-запросов.

Функциональность (Методы):

`protected checkData(data: any): data is IgetProducts` — Выполняет проверку типа: убеждается, что полученные от сервера данные соответствуют ожидаемой структуре IgetProducts. Это пользовательская type guard-функция.

`public async get(uri: string = '/product/'): Promise<IProduct[]>` — Асинхронно запрашивает данные с сервера по указанному адресу. После получения проверяет их с помощью checkData() и возвращает массив товаров. В случае неудачной проверки возвращает пустой массив.

`public async post(uri: string = '/order/', data: object = {})` — Отправляет данные (например, заказ) на сервер. Обрабатывает ответ сервера, возвращая в случае успеха объект с подтверждением заказа или информацию об ошибке.
